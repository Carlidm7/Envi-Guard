Email notifications

Configure EmailJS for alarm messages. The same event may log often; emails are limited to one per 30 minutes per device and variable (e.g. temperature on room A). Send test email always sends immediately.

Email alarm notifications When alarms fire, the app can email via EmailJS (no backend).

You are signed in as an operator. Only an administrator can change these settings.

  1. Create a free EmailJS account at emailjs.com and sign in.
  2. Add an email service (EmailJS → Email Services → Add New Service). Pick Gmail, Outlook, or another provider and connect it.
  3. Create an email template with To = {{to_email}} and optionally {{subject}} / {{message}} in the body.
  4. Copy Public Key, Service ID, and Template ID from EmailJS into the fields below, set Notify (recipient), enable alerts, then Save.
Template ID & example

Template ID is shown in the EmailJS template editor (e.g. template_abc123). Minimal example: 

To email:    {{to_email}}
Subject:     {{subject}}
Content:     {{message}}
Optional template variables

reply_to, alarm_variable, device_id, room, alarm_time
