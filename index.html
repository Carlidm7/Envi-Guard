Email alarm notifications When a new alarm fires (temperature, humidity, or connectivity), the app can email via EmailJS.

You are signed in as an operator. Ask an administrator to sign in and set the notify address and EmailJS keys below.

  1. Create a free EmailJS account at emailjs.com and sign in.
  2. Add an email service (EmailJS → Email Services → Add New Service). Pick Gmail, Outlook, or another provider and connect it. That connection is what actually sends mail (from your mailbox or SMTP).
  3. Create an email template (Email Templates → Create New Template). At minimum, set the template To field to {{to_email}} so messages go to the address you enter below. For subject/body you can use {{subject}} and {{message}} — the app fills those automatically for each alarm.
  4. Copy the three IDs into this dashboard: Account → General shows your Public Key. Your Service ID appears on the service you added; Template ID on the template you created (they look like service_abc, template_xyz).
  5. Notify (recipient email) is who receives alerts (can be your work inbox). It must match an address your EmailJS plan allows (often the same as the connected mailbox on the free tier).
  6. Check Enable email alerts, click Save email settings, then Send test email. If it fails, open the browser developer console (F12) for error details.
Where to find Template ID & a copy-paste template example

In EmailJS: Email Templates → Create New Template (or open an existing one). The Template ID is shown at the top of the editor (e.g. template_abc123) — copy that exact string into the Template ID field below. You do not invent the ID; EmailJS generates it when the template is created.

In the template editor, set the fields like this (minimal working example):

To email:    {{to_email}}
Subject:     {{subject}}

Content (plain text or HTML body):
{{message}}

Optional extra lines using other variables:
Device:      {{device_id}}
Room:        {{room}}
Alarm type:  {{alarm_variable}}
When:        {{alarm_time}}

Attach your Email Service to this template when EmailJS asks (the same Gmail/Outlook service you created). Save the template, then paste its Template ID into Envi-Guard.

Optional template variables

Besides to_email, subject, and message, the app can pass reply_to, alarm_variable, device_id, room, and alarm_time if you want them in custom template fields (e.g. {{alarm_variable}}).
